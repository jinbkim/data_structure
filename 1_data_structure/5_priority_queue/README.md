## 우선순위 큐 ADT (우선순위가 높은 데이터가 먼저 나옴)
- p_queue_init : 우선순위 큐 초기화 
- p_queue_enter : 우선순위 큐에 데이터 추가 
- delete_p_queue : 큐의 우선순위가 높은 데이터 삭제
- p_queue_is_empty : 우선순위 큐가 비었는지 확인
### 힙
- complete binary tree : node가 위에서 아래로,  
  왼쪽에서 오른쪽으로 순서대로 채워진  이진트리
- 힙은 complete binary tree 이면서,  
  모든parent node의 값은 childe node의 값보다 크거나 같아야 함
- 최대 힙 : 루트 노드로 올라갈수록 값이 커지는 완전 이진트리
- 최소 힙 : 루트 노드로 올라갈수록 값이 작아지는 완전 이진트리
### 힙의 데이터 저장과정
- 새로운 데이터를 마지막 위치에 저장
- parent node와 우선순위를 비교
- 새로운 데이터의 우선순위가 높으면 위치 바꾸기
- 비교과정을 반복하여 위치 조정 
### 힙의 데이터 삭제과정
- 루트 노드에 마지막 데이터를 저장
- child node중 우선순위가 높은 node와 우선순위를 비교
- 새로운 데이터의 우선순위가 낮으면 위치 바꾸기
- 비교과정을 반복하여 위치 조정 
### 배열기반 힙
- 인덱스 0인 요소는 비워둠. 
- 힙에 저장된 노드의 개수와 마지막 노드의 인덱스는 일치 
- left child node의 인덱스 = parent node 인덱스 * 2
- right child node의 인덱스 = parent node 인덱스 * 2 + 1
- parent node의 인덱스 = child node 인덱스 / 2
