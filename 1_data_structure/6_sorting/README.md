## 1. 힙정렬
- 힙의 루트 노드에 저장된 값이 우선순위가 가장 높음 
- 힙에 넣었다가 꺼내면 정렬이 완료
### 비교연산 빅오 : nlog2(n), 데이터 이동연산 빅오 : nlog2(n)
## 2. 병합정렬
- 4개의 데이터가 배열에 있다고 가정 (3 2 4 1)
- 하나씩 구분이 될때까지 둘로 나누어 분할 (3   2   4   1)
- 우선순위에 따라 정렬과 결합을 반복  
  (***3*** ***2*** 4 1) -> (2 3 ***4*** ***1***) -> (***2*** ***3*** ***1*** ***4***) ->  (1 2 3 4)
### 비교연산 빅오 : nlog2(n), 데이터 이동연산 빅오 : nlog2(n)
## 3. 퀵정렬
- 4개의 데이터가 배열에 있다고 가정 (***3(left)*** 2 4 ***1(right)***)
#### 1. left 데이터, 중간에 있는 데이터, right 데이터중, 중간값을 가진 데이터를 pivot으로 정함  
- (3 ***2*** 4 1)
- left :0, right :3, pivot index : 1, pivot : 2
#### 2. left 다음 데이터를 low로, right 데이터를 high로 정함
- (3 ***2*** 4 ***1***)
- left :0, right :3, pivot index : 1, pivot : 2, low :1, high : 3
#### 3. pivot index와 left 데이터 교환
- (***3*** ***2*** 4 1) -> (***2*** ***3*** 4 1)
- left :0, right :3, pivot index : 1, pivot : 2, low :1, high : 3
#### 4. 피벗보다 우선순위가 낮은 데이터를 만날때까지, low <= high를 유지하며, low를 오른쪽으로 이동  
- (2 ***3*** 4 1) 
- left :0, right :3, pivot index : 1, pivot : 2, low :1, high : 3
#### 5. 피벗보다 우선순위가 높은 데이터를 만날때까지, left < high를 유지하며, high를 왼쪽으로 이동  
- (2 3 4 ***1***)
- left :0, right :3, pivot index : 1, pivot : 2, low :1, high : 3
#### 6. low <= high 이면, low와 high 위치의 데이터 교환
- (2 ***3*** 4 ***1***) -> (2 ***1*** 4 ***3***)
- left :0, right :3, pivot index : 1, pivot : 2, low :1, high : 3
#### 7. low <= high 를 만족할때까지 4 ~ 6 과정을 반복  
- (2 ***1*** ***4*** 3)
- left :0, right :3, pivot index : 1, pivot : 2, low :2, high : 1
#### 8. left와 high 위치에 있는 데이터 교환  
- (***2*** ***1*** 4 3) -> (***1*** ***2*** 4 3)
- left :0, right :3, pivot index : 1, pivot : 2, low :2, high : 1
#### 9. high 위치를 pivot으로 정함  
- (1 ***2*** 4 3)
- pivot : 1 
#### 10. pivot을 제외한 범위에서 다시 1 ~ 9 과정을 반복하며 정렬
### 비교연산 빅오 : nlog2(n), 데이터 이동연산 빅오 : nlog2(n)
## 4. 기수 정렬
- 4개의 데이터가 배열에 있다고 가정 (321, 23, 212, 6)
#### 1. 버킷(연결리스트 기반 큐) 배열(인덱스 0 ~ 9) 생성
#### 2. 1의 자리의 값과 같은 버킷 배열의 인덱스에 데이터 넣기
- 1(321), 2(212), 3(23), 6(6)
#### 3. 버킷배열을 순서대로 꺼내어 배열에 다시 저장
- (321, 23, 212, 6) -> (321, 212, 23, 6)
#### 4. 10의 자리도 2 ~ 3과정 반복
- 0(6), 1(212), 2(321, 23)
- (321, 212, 23, 6) -> (6, 212, 321, 23)
#### 5. 100의자리도 2 ~ 3과정 반복
- 0(6, 23) 2(212), 3(321)
- (6, 212, 321, 23) -> (6, 23, 212, 321)
### 비교연산 빅오 : n, 데이터 이동연산 빅오 : n
