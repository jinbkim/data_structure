#### 배열기반 리스트 
- 크기가 고정되어 있음.
- 인덱스로 해당원소 접근 하기 때문에, 데이터 조회 시간이 빠름.
- 참조를 위한 주가적인 메모리 할당 필요 x.
- 데이터 삽입/삭제 과정에서 데이터의 이동이 필요하므로, 삽입/삭제가 느림.
#### 연결리스트
- 크기가 고정되어 있지 않음.
- 자기 다음에 어떤 원소인지 만을 기억하기 때문에, 데이터 조회 시간이 느림.
- 참조를 위한 추가 메모리 할당이 필요하고, 메모리 재사용이 가능함. 
#### 여러가지 연결리스트
- 원형 연결리스트 : 꼬리노드가 머리노드를 가리키는 연결리스트
- 양방향 연결리스트 : 각각 노드가 한쪽방향이 아닌 양쪽방향으로의 연결된 리스트
#### 연결리스트 사용법 
- 데이터의 개수가 정해져있고, 데이터의 삽입과 삭제가 많지 않으면  -> 배열기반 리스트
- 데이터의 개수가 정해져 있지않고, 데이터의 삽입과 삭제가 많으면  -> 연결리스트
