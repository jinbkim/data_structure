## 2. 자료구조
### 2.1) 배열
- 연속된 메모리 공간을 차지하는 같은 데이터형들의 집합
#### 함수인자로 배열 넘기는법
- char a[2] -> char a[]
- int b[3][4] -> int b[][4]
- double c[5][6][7] -> double c[][6][7]
### 2.2) 연결리스트
- 실제로 정보를 담고있는 하나의 단위인 노드와 노드의 위치를 저장하는 리스트로 구성
- 배열과 다르게 동적인 자료구조
- 머리와 꼬리를 정의하여 코드의 일관성을 유지 할수있음
#### 단순 연결리스트
- 정보를 저장하는 노드와 다음노드를 가리키는 링크로 구성됨
#### 원형 연결리스트
- 꼬리노드가 머리노드를 가리키는 연결리스트
#### 이중 연결리스트
- 단순 연결리스트에 이전노드를 가리키는 링크가 추가된것
### 2.3) 스택
- last in, first out
- push : 스택상단의 데이터를 집어넣는 동작
- pop : 스택상단의 데이터를 빼내는 동작
#### 배열기반 스택
- 메모리 낭비가 우려됨
- 인덱스를 이용하여 메모리에 바로 접근하므로 속도가 빠름
#### 연결리스트기반 스택
- 메모리를 효율적으로 사용가눙
- 노드를 이동하면서 메모리에 접근하므로 속도가 느림
### 2.4) 큐
- first in, first out
- put : 큐의 뒤에서 데이터를 집어넣는 동작
- get : 큐의 앞에서 데이터를 빼내는 동작
